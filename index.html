Review sesi 8 - Builder Pattern

Builder Pattern adalah salah satu design pattern yang digunakan untuk membuat object yang kompleks secara bertahap (step-by-step). Pattern ini bertujuan untuk memisahkan proses pembuatan object dari representasinya, sehingga object yang sama dapat dibuat dengan cara yang berbeda.

Pengertian Extension Methods

Extension Methods merupakan fitur dalam Dart yang digunakan untuk menambahkan method atau fungsi baru ke tipe data yang sudah ada tanpa harus mengubah class aslinya. Dengan menggunakan extension, developer dapat memperluas kemampuan dari tipe data bawaan seperti String, List, atau DateTime, maupun custom class, sehingga kode menjadi lebih fleksibel dan mudah digunakan. Extension dideklarasikan menggunakan sintaks extension NamaExtension on TipeData, dan dapat berisi method, getter, setter, maupun operator. Penggunaan extension membuat penulisan kode menjadi lebih natural karena method dapat dipanggil langsung dari object, seolah-olah merupakan bagian dari class tersebut.

🔹 Masalah tanpa Extension Method

Tanpa menggunakan Extension Methods, penambahan fungsi pada tipe data harus dilakukan melalui utility class atau helper function. Cara ini membuat pemanggilan method menjadi kurang natural karena harus dipanggil melalui nama class, bukan langsung dari object. Selain itu, kode menjadi lebih panjang, kurang rapi, dan sulit dibaca, terutama jika fungsi yang digunakan sering dipanggil. Pendekatan ini juga memisahkan fungsi dari tipe datanya, sehingga mengurangi keterbacaan dan kenyamanan dalam penggunaan kode.
// Nama : Nabilah cahya suci // NIM : 251410018 // Kelas : SI2A class StringUtils { static String capitalize(String text) { if (text.isEmpty) return text; return text[0].toUpperCase() + text.substring(1); } static bool isEmail(String text) { return text.contains('@'); } } void main() { var name = 'john'; var capitalized = StringUtils.capitalize(name); var isEmail = StringUtils.isEmail('test@mail.com'); print('Original name: $name'); print('Capitalized name: $capitalized'); print('Is "test@mail.com" an email? $isEmail'); }

🔹 Solusi dengan Extension Method

Ini adalah solusi dari slide sebelumnya. Dengan Extension Methods, bisa menambahkan fitur baru ke tipe String tanpa mengubah kode aslinya.
// Nama : Nabilah cahya suci // NIM : 251410018 // Kelas : SI2A extension StringExtensions on String { String capitalize() { if (isEmpty) return this; return this[0].toUpperCase() + substring(1); } bool get isEmail => contains('@'); String reverse() { return split('').reversed.join(); } } void main() { var name = 'john'; var capitalized = name.capitalize(); var isEmail = 'test@mail.com'.isEmail; var reversed = 'hello'.reverse(); print('Original name: $name'); print('Capitalized name: $capitalized'); print('Is "test@mail.com" an email? $isEmail'); print('Reversed "hello": $reversed'); }

🔹 Sintaks Extension Method

Sintaks Extension Methods digunakan untuk menambahkan fungsi ke tipe data tertentu dengan format extension NamaExtension on TipeData. Pada bagian ini, NamaExtension merupakan nama dari extension yang dibuat, sedangkan TipeData adalah tipe data yang akan ditambahkan method. Di dalam extension, dapat dituliskan method, getter, setter, maupun operator. Dengan sintaks ini, method yang dibuat dapat digunakan langsung pada object dari tipe data tersebut, seolah-olah merupakan bagian dari class aslinya.
// Nama : Nabilah cahya suci // NIM : 251410018 // Kelas : SI2A extension DateTimeExtensions on DateTime { String toFormattedString() { return '$day/$month/$year'; } bool get isToday { var now = DateTime.now(); return year == now.year && month == now.month && day == now.day; } DateTime operator +(Duration duration) { return add(duration); } } void main() { final now = DateTime.now(); print('Current date: ${now.toFormattedString()}'); final specificDate = DateTime(2023, 10, 26); print('Specific date: ${specificDate.toFormattedString()}'); final today = DateTime.now(); final tomorrow = today.add(Duration(days: 1)); print('Is today? ${today.isToday}'); print('Is tomorrow today? ${tomorrow.isToday}'); final futureDate = now + Duration(days: 5); print('Future date: ${futureDate.toFormattedString()}'); }

🔹 Tipe yang bisa di-Extend

Extension Methods dapat digunakan pada berbagai tipe data dalam Dart, baik tipe bawaan maupun tipe buatan sendiri. Tipe yang bisa di-extend meliputi tipe data bawaan seperti String, int, double, List, dan DateTime, serta custom class yang dibuat oleh programmer. Selain itu, extension juga mendukung generic type seperti List, sehingga dapat digunakan secara fleksibel untuk berbagai tipe data. Dengan demikian, hampir semua tipe data di Dart dapat diperluas fungsinya menggunakan extension methods.
// Nama : Nabilah cahya suci // NIM : 251410018 // Kelas : SI2A extension IntListExtensions on List { int get sum => fold(0, (total, item) => total + item); double get average => isEmpty ? 0 : sum / length; } void main() { var numbers = [1, 2, 3, 4, 5]; print(numbers.sum); print(numbers.average); }

🔹 Extension untuk Null Safety

Extension untuk Null Safety digunakan untuk menangani tipe data yang bersifat nullable (memiliki kemungkinan bernilai null), seperti String?. Dengan extension ini, kita dapat menambahkan method atau getter yang membantu menghindari error akibat null, seperti memberikan nilai default atau melakukan pengecekan null dengan lebih mudah. Extension ini biasanya digunakan untuk membuat kode lebih aman, ringkas, dan mudah dibaca saat bekerja dengan null safety di Dart.
// Nama : Nabilah cahya suci // NIM : 251410018 // Kelas : SI2A extension NullableStringExtensions on String? { String orEmpty() => this ?? ''; bool get isNullOrEmpty => this == null || this!.isEmpty; bool get isNotNullOrEmpty => !isNullOrEmpty; String withDefault(String defaultValue) => this ?? defaultValue; } void main() { String? name; print(name.orEmpty()); print(name.isNullOrEmpty); print(name.isNotNullOrEmpty); print(name.withDefault('Guest')); }

penggunaan Extension Methods untuk membuat berbagai fungsi tambahan (utilities) pada tipe data bawaan Dart seperti String, List, dan DateTime. Dengan extension, utility function yang sebelumnya dibuat dalam bentuk helper class dapat langsung ditempelkan pada tipe data terkait, sehingga penggunaannya menjadi lebih sederhana dan natural. Contohnya seperti menambahkan fungsi capitalize() pada String, sum dan average pada List, atau format tanggal pada DateTime. Pendekatan ini membuat kode lebih ringkas, mudah dibaca, dan terorganisir karena fungsi berada dekat dengan tipe data yang digunakan.
// Nama : Nabilah cahya suci // NIM : 251410018 // Kelas : SI2A extension NumberExtensions on num { String toRupiah() { return 'Rp${toStringAsFixed(0).replaceAllMapped( RegExp(r'(\d{1,3})(?=(\d{3})+(?!\d))'), (match) => '${match[1]}.', )}'; } bool isBetween(num min, num max) { return this >= min && this <= max; } num power(int exponent) { var result = this; for (var i = 1; i < exponent; i++) { result *= this; } return result; } } void main() { var price = 1500000; print(price.toRupiah()); print(5.isBetween(1, 10)); print(2.power(3)); }

🔹 Generic Extension

Generic Extension adalah extension yang dapat digunakan untuk berbagai tipe data dengan menggunakan generic (). Dengan generic, extension tidak hanya berlaku untuk satu tipe tertentu, tetapi bisa digunakan secara fleksibel untuk banyak tipe data. Biasanya digunakan pada collection seperti List, sehingga method yang dibuat dapat bekerja untuk semua jenis isi list. Generic extension membantu membuat kode lebih reusable, fleksibel, dan tidak perlu membuat extension terpisah untuk setiap tipe data.
// Nama : Nabilah cahya suci // NIM : 251410018 // Kelas : SI2A extension ListExtensions on List { T? getOrNull(int index) { if (index < 0 || index >= length) return null; return this[index]; } List> chunk(int size) { var chunks = >[]; for (var i = 0; i < length; i += size) { chunks.add( sublist(i, i + size > length ? length : i + size), ); } return chunks; } List distinct() { var result = []; for (var item in this) { if (!result.contains(item)) { result.add(item); } } return result; } } void main() { var numbers = [1, 2, 3, 4, 5, 6, 7]; print(numbers.chunk(3)); print([1, 2, 2, 3, 3].distinct()); }

🔹 Unnamed Extensions

Unnamed Extensions adalah extension yang dibuat tanpa memberikan nama pada extension tersebut. Extension ini hanya berlaku di dalam file tempat extension tersebut didefinisikan, sehingga tidak dapat digunakan atau diakses dari file lain. Unnamed extension biasanya digunakan untuk kebutuhan internal atau penggunaan terbatas agar tidak terjadi konflik nama dengan extension lain. Dengan demikian, unnamed extension membantu menjaga kode tetap aman dan terorganisir tanpa menambah kompleksitas pada scope global.
// Nama : Nabilah cahya suci // NIM : 251410018 // Kelas : SI2A extension on DateTime { String get toYYYYMMDD { return '$year-${month.toString().padLeft(2, '0')}-${day.toString().padLeft(2, '0')}'; } DateTime get startOfDay { return DateTime(year, month, day); } } void main() { var today = DateTime.now(); print(today.toYYYYMMDD); print(today.startOfDay); }

🔹 Konflik Nama Extension

Konflik Nama Extension terjadi ketika terdapat dua atau lebih extension yang memiliki nama method yang sama pada tipe data yang sama. Hal ini menyebabkan ambiguitas saat pemanggilan method karena Dart tidak dapat menentukan extension mana yang harus digunakan. Untuk mengatasi konflik ini, dapat digunakan beberapa cara seperti menggunakan keyword hide untuk menyembunyikan method tertentu saat import, atau menggunakan alias (as) pada import sehingga pemanggilan method dapat dilakukan secara spesifik. Dengan cara ini, konflik dapat dihindari dan kode tetap jelas serta terstruktur.
// Nama : Nabilah cahya suci // NIM : 251410018 // Kelas : SI2A extension StringCaseExtensions on String { String capitalize() { if (isEmpty) return this; return this[0].toUpperCase() + substring(1); } } extension TextUtilsExtensions on String { String capitalize() { return toUpperCase(); // implementasi berbeda } } void main() { var text = 'hello'; // Panggil eksplisit untuk hindari konflik print(StringCaseExtensions(text).capitalize()); // Hello print(TextUtilsExtensions(text).capitalize()); // HELLO }

🔹 Extension untuk Custom Classes

Extension untuk Custom Classes digunakan untuk menambahkan method atau properti ke class yang dibuat sendiri tanpa harus mengubah class aslinya. Dengan extension, kita dapat memperluas fungsi dari class tersebut, seperti menambahkan fitur perhitungan, formatting, atau pengecekan kondisi. Hal ini membuat kode lebih rapi, terorganisir, dan mudah dikembangkan karena logic tambahan tidak dicampur langsung di dalam class utama. Extension pada custom class juga membantu menjaga prinsip clean code dengan memisahkan tanggung jawab antara data dan fungsi tambahan.
// Nama : Nabilah cahya suci // NIM : 251410018 // Kelas : SI2A class Product { final String name; final double price; Product(this.name, this.price); } extension DoubleRupiahFormatter on double { String toRupiah() { final intValue = toInt(); String numberString = intValue.toString(); String formatted = ''; int counter = 0; for (int i = numberString.length - 1; i >= 0; i--) { formatted = numberString[i] + formatted; counter++; if (counter % 3 == 0 && i != 0) { formatted = '.' + formatted; } } return 'Rp$formatted'; } } extension ProductExtensions on Product { Product withDiscount(double percentage) { var newPrice = price * (1 - percentage / 100); return Product(name, newPrice); } String get displayInfo => '$name - ${price.toRupiah()}'; bool get isExpensive => price > 1000000; } void main() { var laptop = Product('Laptop', 15000000); var discounted = laptop.withDiscount(10); print(discounted.displayInfo); print(laptop.isExpensive); }

🔹 Praktik 2 – Validator Extensions

Extension Methods untuk membuat fungsi validasi (validator) pada tipe data seperti String dan int. Dengan extension, proses validasi dapat dilakukan langsung dari object, sehingga penulisan kode menjadi lebih sederhana dan mudah dibaca. Contohnya seperti pengecekan email, nomor telepon, password kuat, atau validasi angka seperti usia dan bilangan genap/ganjil. Pendekatan ini membuat kode lebih rapi karena fungsi validasi tidak perlu dipisahkan dalam utility class, serta dapat digunakan kembali dengan mudah di berbagai bagian program.
// Nama : Nabilah cahay suci // NIM : 251410018 // Kelas : SI2A extension StringValidators on String { bool get isEmail { return RegExp(r'^[\w-\.]+@([\w-]+\.)+[\w-]{2,4}$') .hasMatch(this); } bool get isPhoneNumber { return RegExp(r'^[0-9]{10,13}$').hasMatch(this); } bool get isStrongPassword { return length >= 8 && contains(RegExp(r'[A-Z]')) && contains(RegExp(r'[a-z]')) && contains(RegExp(r'[0-9]')); } bool get isUrl { try { Uri.parse(this); return true; } catch (_) { return false; } } } extension IntValidators on int { bool get isAdultAge => this >= 18; bool get isPositive => this > 0; bool get isEven => this % 2 == 0; bool get isOdd => this % 2 != 0; } void main() { print('test@example.com'.isEmail); print(25.isAdultAge); print('Pass1234'.isStrongPassword); print(10.isEven); }

Kesimpulan

Extension Methods adalah fitur Dart untuk menambahkan fungsi pada tipe data tanpa mengubah class asli. Extension membuat kode lebih ringkas, natural, dan mudah dibaca karena method dipanggil langsung dari object. Extension mendukung method, getter, setter, operator, serta bisa digunakan pada tipe bawaan, custom class, generic, dan null safety. Penggunaannya membantu membuat kode lebih rapi dan reusable, namun harus menghindari overuse dan konflik nama agar tetap terstruktur dengan baik.
